# Phase 3.1.5: Preset System - Design Specification

> **Status:** Design Approved
> **Created:** 2026-03-23
> **Phase:** 3.1.5 (Video Transitions - Preset System)

## Overview

Implement a comprehensive preset management system that enables users to create, save, organize, and share custom transition presets. Users can save their current transition configurations as named presets, organize them into folders, mark favorites, and share presets via import/export.

## Goals

- **User Creations:** Enable users to save custom transition configurations as reusable presets
- **Organization:** Provide folder-based organization and favorites for easy access
- **Sharing:** Support import/export for sharing presets with others or between projects
- **Visual Discovery:** Display preset thumbnails so users can identify transitions at a glance
- **Persistence:** Custom presets persist across application launches

## Architecture

### Design Philosophy

**Hybrid UI Pattern**

Following the pattern from professional video editing tools (Final Cut Pro, DaVinci Resolve), the preset system uses a hybrid approach:
- **Quick save** in the inspector for immediate workflow integration
- **Dedicated library window** for comprehensive management features

**Layered Architecture**

```
UI Layer (Inspector + Manager Window)
    ↓
Business Logic (PresetLibrary)
    ↓
Persistence (TransitionPresetStorage)
    ↓
Disk Storage (JSON files + thumbnails)
```

**Preview Generation**

Thumbnails use the existing `TransitionRenderer` infrastructure from Phase 3.1.3, rendered at the transition midpoint (50% progress) to show a representative preview.

**Storage Location:**
```
~/Library/Application Support/OpenScreen/TransitionPresets/

Adapts the existing `PresetStorage` pattern from Effects, using JSON encoding and percent-encoded filenames.

    "softness": 0.5,

```

### 2. PresetLibrary

**File:** `Sources/native-macos/Transitions/Presets/PresetLibrary.swift`

**Responsibilities:**
- Manage in-memory collection of all presets (built-in + custom)
- Maintain folder organization
- Track favorites
- Validate preset names and operations
- Handle import/export

**Interface:**

```swift
@MainActor
final class PresetLibrary: ObservableObject {

    /// All presets (built-in + custom)
    private(set) var allPresets: [TransitionPreset] = []

    /// Folder organization
    private(set) var folders: Set<String> = ["My Transitions"]

    /// Initialize library with built-in presets
    init()

    /// Load custom presets from storage
    func loadCustomPresets() throws

    /// Save a new custom preset
    func savePreset(
        name: String,
        folder: String,
        transition: TransitionClip,
        isFavorite: Bool
    ) throws

    /// Delete a custom preset
    func deletePreset(_ preset: TransitionPreset) throws

    /// Update preset metadata
    func updatePreset(_ preset: TransitionPreset, name: String? = nil, folder: String? = nil)

    /// Toggle favorite status
    func toggleFavorite(_ preset: TransitionPreset)

    /// Get presets in a specific folder
    func presetsInFolder(_ folder: String) -> [TransitionPreset]

    /// Get favorite presets
    func favoritePresets() -> [TransitionPreset]

    /// Import preset from file
    func importPreset(from url: URL) throws

    /// Export preset to file
    func exportPreset(_ preset: TransitionPreset, to url: URL) throws

    /// Validate preset name
    func validatePresetName(_ name: String) throws
}
```

**Folder Metadata:**
- Stored as `TransitionPreset` extension property or separate metadata file
- Preset belongs to one folder
- "All" and "Favorites" are virtual folders (views, not storage)

## Data Flow

### Saving a Custom Preset

```
User clicks "Save as Preset..." in inspector
→ SavePresetSheet opens with current transition settings pre-populated
→ User enters name "My Dramatic Wipe" and selects folder
→ Click Save
→ PresetLibrary.validatePresetName() checks for duplicates/built-in names
→ PresetLibrary.savePreset() creates TransitionPreset with current settings
→ PresetLibrary.presetStorage.savePreset(preset)
→ Background: PresetPreviewRenderer.generateThumbnail(for: preset)
→ Thumbnail saved to disk via storage.saveThumbnail()
→ Preset added to in-memory library
→ Inspector's preset menu updated to show new preset
```

### Loading on Application Launch

```
Application launches
→ PresetLibrary.initialized()
→ Built-in presets loaded from BuiltInPresets.presets
→ PresetLibrary.loadCustomPresets() called
→ TransitionPresetStorage.loadCustomPresets() loads all JSON files
→ Merge into single library with folder metadata
→ Thumbnail cache initialized
→ Inspector's preset menu shows built-ins + custom presets organized by folder
→ PresetManagerWindow shows all presets
```

### Import/Export Workflow

**Export:**
```
User selects preset(s) in PresetManagerWindow
→ Click "Export Selected..."
→ File save dialog appears
→ User chooses location and filename
→ PresetLibrary.exportPreset() encodes preset as JSON
→ Saved to disk
→ Can be shared via email, cloud storage, etc.
```

**Import:**
```
User selects "Import..." from toolbar
→ File open dialog appears
→ User selects JSON file (could be from another user or backup)
→ PresetLibrary.importPreset() validates JSON structure
→ Checks for name collision with existing presets
→ If collision: Dialog asks "Replace, Rename, or Cancel?"
→ Replace option: Overwrites existing preset (if custom, cannot replace built-in)
→ Rename option: Auto-generates "Preset Name (Imported)" or prompts user for new name
→ Cancel option: Aborts import
→ If name collision with built-in: Always force rename (built-ins are read-only)
→ Thumbnail: Copied from imported file if present, otherwise regenerated on-demand
→ Preset added to library (user can select folder, defaults to "Imported")
```

### Thumbnail Generation

```
Preset first viewed in browser (PresetManagerWindow or Inspector)
→ Check if thumbnail exists on disk (storage.loadThumbnail())
→ If exists: Load from disk and display
→ If not:
    → PresetPreviewRenderer.render(preset)
    → Creates test frames: leading (red), trailing (blue)
    → Gets renderer from TransitionRenderContext
    → Renders at progress = 0.5
    → Converts CIImage to PNG
    → Saves to Thumbnails/ folder via storage.saveThumbnail()
    → Displays to user
→ Subsequent views load from cache
```

## Error Handling

### Preset Validation

**Invalid preset names:**
- Empty names → `PresetError.emptyName`
- Names colliding with built-ins → `PresetError.reservedName`
- Duplicate names → `PresetError.duplicateName`

**Invalid preset data:**
- Duration outside valid range (0.1-5.0s) → `TransitionError.invalidDuration`
- Parameters don't match transition type → Should not occur (UI prevents this)

### Storage Errors

**Save failures:**
- Permission denied → `PresetError.saveFailed(reason:)`
- Disk full → `PresetError.diskFull`
- Invalid characters in filename → Auto-sanitize using percent-encoding

**Load failures:**
- Corrupted JSON file → Log warning, skip file, continue loading others
- Missing thumbnail → Regenerate on-demand (not an error)

### Import/Export Errors

**Import failures:**
- Invalid JSON → `PresetError.invalidFile`
- Version mismatch → `PresetError.incompatibleVersion`
- Name collision → Show dialog with options: Replace, Rename, Cancel

### Thumbnail Generation Failures

**Rendering failures:**
- Renderer returns nil → Use fallback icon based on transition type
- Crash during render → Log error, use fallback icon, don't retry

## Known Limitations

### For Phase 3.1.5 (MVP)

- **Flat folder structure** - No nested folders/subfolders (can be added in Phase 3.2)
- **No preset search** - For small libraries (< 100 presets), scrolling is sufficient
- **No preset sharing service** - Only file-based import/export
- **Single thumbnail per preset** - No multiple preview points or video loops
- **No preset versioning** - Can't track changes or revert to previous versions

### Postponed to Future Phases

- **Preset marketplace/sharing platform** - Cloud-based preset library
- **Collaborative preset libraries** - Team sharing, version control
- **Preset analytics** - Most used presets, usage patterns
- **Advanced thumbnail generation** - Video loop previews, multiple preview points
- **Preset collections/themes** - Bundled preset packages
- **AI-powered preset suggestions** - Recommend presets based on content

## Future Enhancements

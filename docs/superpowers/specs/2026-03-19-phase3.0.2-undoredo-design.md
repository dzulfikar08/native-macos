# Phase 3.0.2: Clip Operations and Undo/Redo - Design Specification

**Status:** Design
**Date:** 2026-03-19
**Authors:** User + Claude Sonnet 4.6
**Related:** Phase 3.0.1 Foundation (this phase builds upon)

---

## Overview

Phase 3.0.2 implements a comprehensive undo/redo system that wraps all editor operations in reversible command objects. This enables professional-grade editing workflows where users can freely experiment and correct mistakes without fear of losing work.

**Key Goals:**
- Wrap all editor operations (clips, mode switching, markers, effects) in undoable commands
- Provide smart coalescing for rapid repeated operations
- Maintain operation history with hybrid limits (count + time)
- Present rich contextual information about undo/redo actions
- Support standard keyboard shortcuts (Cmd+Z, Cmd+Shift+Z)

**Timeline:** 2-3 weeks
**Dependencies:** Phase 3.0.1 Foundation (complete)

### Key Design Decisions

1. **Operation Wrapping:** Each ClipManager method wrapped in dedicated operation class
2. **State Capture:** Operations capture state *before* modification for undo
3. **Smart Coalescing:** Operations within 1-second window with same type/target merge
4. **Hybrid History:** Soft limit of 100 operations, preserve last 10 minutes always
5. **Full Context:** Rich descriptions with clip names, parameters, before/after values
6. **Comprehensive Scope:** All editor operations (clips, mode, markers, effects)

## Section 2: Concrete Operation Implementations

### Operation Structure

All concrete operations follow this pattern:

```swift
class SpecificOperation: BaseClipOperation {
    // Capture state before modification
    private let previousState: StateSnapshot
    private let operationDetails: OperationDetails

    init(..., editorState: EditorState, clipManager: ClipManager) {
        // Capture current state before changes
        self.previousState = captureCurrentState()
        self.operationDetails = ...
        super.init(description: "...", editorState: editorState, clipManager: clipManager)
    }

    func execute() throws {
        // Apply changes using ClipManager
        try clipManager?.performOperation(...)
    }

    func undo() throws {
        // Restore to previousState
        restoreState(previousState)
    }

    func redo() throws {
        // Re-apply the operation
        try execute()
    }

    // Rich description for UI
    override var description: String {
        let details = operationDetails
        return "Operation: \(details.clipName), from: \(details.oldValue), to: \(details.newValue)"
    }
}
```

### Operation Categories

#### 1. Clip Modifications (6 operations)

**SplitClipOperation**
- **Captures:** Original clip ID, name, timeRangeInSource, timeRangeInTimeline
- **Execute:** Calls `clipManager.splitClip(clipID:at:)`
- **Undo:** Removes both clips, restores original clip
- **Description:** "Split Clip: {name} at {timestamp}"

**TrimClipOperation**
- **Captures:** Original timeRangeInSource, original timeRangeInTimeline
- **Execute:** Calls `clipManager.trimClip(clipID:to:)`
- **Undo:** Restores original time ranges
- **Description:** "Trim Clip: {name} from {oldDuration} to {newDuration}"

**MoveClipOperation**
- **Captures:** Original trackID, original timeRangeInTimeline
- **Execute:** Calls `clipManager.moveClip(clipID:to:on:ripple:)`
- **Undo:** Moves clip back to original track and position
- **Description:** "Move Clip: {name} from {oldTrack} @ {oldPosition} to {newTrack} @ {newPosition}"

**DeleteClipOperation**
- **Captures:** Full clip state (all properties), original track index
- **Execute:** Calls `clipManager.deleteClip(clipID:ripple:)`
- **Undo:** Re-creates clip and re-adds to track
- **Description:** "Delete Clip: {name}"

**DuplicateClipOperation**
- **Captures:** Track state before duplication, clip count
- **Execute:** Calls `clipManager.duplicateClip(clipID:to:)`
- **Undo:** Removes duplicated clip
- **Description:** "Duplicate Clip: {name} to {position}"

**ChangeSpeedOperation**
- **Captures:** Original speed, original timelineDuration
- **Execute:** Calls `clipManager.changeClipSpeed(clipID:to:)`
- **Undo:** Restores original speed
- **Description:** "Change Speed: {name} from {oldSpeed}x to {newSpeed}x (duration: {oldDuration} → {newDuration})"

#### 2. Timeline Operations (1 operation)

**SwitchTimelineModeOperation**
- **Captures:** Previous TimelineEditMode, clipTracks array (if switching to single-asset), TimelineTrack array (if switching to multi-clip)
- **Execute:** Converts timeline state, updates `editorState.timelineEditMode`
- **Undo:** Restores previous mode and track arrays
- **Description:** "Switch Timeline Mode: {fromMode} → {toMode}"
- **Note:** Destructive - shows confirmation dialog before executing

#### 3. Marker Operations (2 operations)

**AddMarkerOperation**
- **Captures:** chapterMarkers array count before addition
- **Execute:** Adds marker to `editorState.chapterMarkers`
- **Undo:** Removes marker at added index
- **Description:** "Add Marker: {name} at {timestamp}"

**DeleteMarkerOperation**
- **Captures:** Marker being deleted (all properties)
- **Execute:** Removes marker from `editorState.chapterMarkers`
- **Undo:** Re-adds marker with original properties
- **Description:** "Delete Marker: {name} at {timestamp}"

#### 4. Effect Operations (2 operations)

**ApplyEffectOperation**
- **Captures:** Clip's effectStack before applying, current effect count
- **Execute:** Adds effect to clip's videoEffects or audioEffects array
- **Undo:** Removes effect from stack
- **Description:** "Apply Effect: {effectType} to {clipName}"

**RemoveEffectOperation**
- **Captures:** Effect being removed (type, parameters, position in stack)
- **Execute:** Removes effect from clip's effect stack
- **Undo:** Re-adds effect with original parameters
- **Description:** "Remove Effect: {effectType} from {clipName}"

### State Snapshot Pattern

Operations capture minimal state needed for undo:

- **Primitives:** Float, Bool, Int captured directly
- **Complex Types:** CMTime, CMTimeRange captured as struct copies
- **AVAsset:** Captured as asset URL, recreated on undo
- **Arrays:** Captured as count + identifiers, full reconstruction on undo
- **Clips:** Captured by UUID reference, re-fetched on undo

## Section 3: ClipUndoManager Implementation

### Location
`Sources/native-macos/UndoRedo/ClipUndoManager.swift`

### Interface

```swift
@MainActor
class ClipUndoManager {
    weak var editorState: EditorState?
    weak var clipManager: ClipManager?

    // History management
    private(set) var undoStack: [ClipOperation] = []
    private(set) var redoStack: [ClipOperation] = []

    // Configuration
    var coalescingStrategy: CoalescingStrategy = .smart
    var historyLimit: HistoryLimit = .hybrid(maxOps: 100, timeWindow: 600)

    // Operation execution
    func executeOperation(_ operation: ClipOperation) throws
    func undo() throws
    func redo() throws
    func clearHistory()

    // Query state
    var canUndo: Bool { !undoStack.isEmpty }
    var canRedo: Bool { !redoStack.isEmpty }

    // Descriptions for UI
    var undoDescription: String? { undoStack.last?.description }
    var redoDescription: String? { redoStack.last?.description }
}
```

### Coalescing Strategy

```swift
enum CoalescingStrategy {
    case none                                    // No coalescing
    case timeWindow(TimeInterval)                // Merge operations within time window
    case sameTypeAndTarget                       // Merge same operation type on same target
    case smart                                   // Time window + type + target (default)
}

struct CoalescingConfig {
    var timeWindow: TimeInterval = 1.0  // 1 second default
    var enabled: Bool = true
}
```

### History Limit

```swift
enum HistoryLimit {
    case unlimited                              // Keep all operations
    case fixedCount(Int)                        // Max N operations
    case timeWindow(TimeInterval)               // Keep operations within time window
    case hybrid(maxOps: Int, timeWindow: TimeInterval)  // Soft limit + time window (default)
}
```

### Execute Flow

1. User calls `executeOperation(_:)`
2. Apply coalescing if enabled:
   - Check if last operation on undoStack matches type/target
   - Check if within coalescing time window (≤ 1.0s)
   - If both match, merge into composite operation
3. Call operation's `execute()`
4. On success: push to undoStack, clear redoStack
5. Enforce history limit:
   - If count > maxOps: trim oldest operations
   - If oldest operation > timeWindow: trim it
6. Post notification: `.undoStackDidChange`

### Undo Flow

1. User calls `undo()`
2. Pop operation from undoStack
3. Call operation's `undo()`
4. On success: push to redoStack
5. Post notification: `.undoStackDidChange`
6. On failure: clear redoStack (state uncertain), re-throw error

### Redo Flow

1. User calls `redo()`
2. Pop operation from redoStack
3. Call operation's `redo()`
4. On success: push to undoStack
5. Post notification: `.undoStackDidChange`
6. On failure: clear redoStack, re-throw error

### History Enforcement

```swift
private func enforceHistoryLimit() {
    switch historyLimit {
    case .unlimited:
        return
    case .fixedCount(let maxOps):
        while undoStack.count > maxOps {
            undoStack.removeFirst()
        }
    case .timeWindow(let window):
        let cutoff = Date().addingTimeInterval(-window)
        undoStack.removeAll { op in op.timestamp < cutoff }
    case .hybrid(let maxOps, let timeWindow):
        // Apply both limits
        while undoStack.count > maxOps {
            undoStack.removeFirst()
        }
        let cutoff = Date().addingTimeInterval(-timeWindow)
        undoStack.removeAll { op in op.timestamp < cutoff }
    }
}
```

## Section 5: Error Handling and Edge Cases

### Operation Execution Errors

When `execute()` fails:

1. **State unchanged** - Operation didn't modify anything
2. **Error propagated** - Original error thrown to caller
3. **No history change** - Failed operations not added to undoStack

```swift
func executeOperation(_ operation: ClipOperation) throws {
    do {
        try operation.execute()
        undoStack.append(operation)
        redoStack.removeAll()
        enforceHistoryLimit()
        postUndoChangeNotification()
    } catch {
        // Operation failed, state unchanged
        throw error
    }
}
```

### Undo Failures

When `undo()` fails (rare):

1. **Operation remains on undoStack** - Can retry undo
2. **RedoStack cleared** - Can't redo after failed undo
3. **User notified** - Show error with recovery suggestion
4. **Operation marked as dirty** - Flag for investigation

```swift
func undo() throws {
    guard let operation = undoStack.last else {
        throw UndoError.nothingToUndo
    }

    do {
        try operation.undo()
        undoStack.removeLast()
        redoStack.append(operation)
        postUndoChangeNotification()
    } catch {
        // Undo failed - clear redo stack since state is uncertain
        redoStack.removeAll()
        throw error
    }
}
```

### Edge Cases

#### 1. State Deserialization

**Issue:** Operations not serializable (closures, AVAsset references)

**Solution:** Clear undo/redo history on project load/save

```swift
func loadProject(from url: URL) throws {
    // Load project...
    undoManager.clearHistory()
}
```

#### 2. Coalescing Edge Cases

**Boundary conditions:**
- Operation at exactly 1.0 second boundary → Include (≤ 1.0s)
- Mixed operation types in coalescing window → Keep separate
- User performs undo during coalescing window → Break window, start new

#### 3. Memory Management

**AVAsset in operations:**
- Use weak references where possible
- Or capture asset URL, recreate on undo

**Large undo history:**
- Enforce hybrid limit strictly
- Monitor memory usage, trim if needed

**Coalesced operations:**
- Limit sub-operation count (max 100)
- If exceeded, break into separate operations

#### 4. Thread Safety

- All operations already @MainActor
- UndoManager @MainActor
- No additional synchronization needed

#### 5. Undo After Mode Switch

**Issue:** Switching timeline modes clears clipTracks ↔ TimelineTrack

**Solution:**
- Show warning: "Switching modes will clear undo history. Continue?"
- User confirms → clear history, execute mode switch
- User cancels → abort mode switch

```swift
func switchTimelineMode(to newMode: TimelineEditMode) async throws {
    if undoManager.canUndo {
        let confirmation = await showConfirmationAlert(
            title: "Clear Undo History?",
            message: "Switching modes will clear undo history. Continue?"
        )
        guard confirmation else { return }
    }

    undoManager.clearHistory()
    // Perform mode switch...
}
```

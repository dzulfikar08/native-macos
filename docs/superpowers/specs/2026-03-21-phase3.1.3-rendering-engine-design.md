# Phase 3.1.3: Rendering Engine - Design Specification

> **Status:** Design Approved
> **Created:** 2026-03-21
> **Phase:** 3.1.3 (Video Transitions - Rendering Engine)

## Overview

Implement CoreImage-based transition rendering for real-time preview in the timeline. This phase focuses on using native CIFilter implementations for common transition types, establishing the rendering pipeline before adding custom Metal shaders in a future phase.

## Goals

- **Pragmatic approach:** Leverage Apple's CIFilter implementations where possible
- **Smooth preview:** Enable real-time transition preview during timeline scrubbing/playback
- **Extensible architecture:** Design renderer to support Metal shaders in future phases
- **Performance:** Target 60fps preview through filter caching and CIContext reuse
- **Graceful degradation:** Clearly indicate which transitions require Metal shaders

## Architecture

### Design Philosophy

**Standalone Renderer Pattern**

Following the established `PipCompositor` pattern, `TransitionRenderer` is a standalone, testable component that:
- Takes two CIImage frames as input
- Returns a blended CIImage as output
- Manages its own CIContext for GPU rendering
- Caches compiled CIFilters for performance

**Progress-Based Animation**

Transitions are time-based animations driven by a `progress` parameter (0.0 to 1.0):
- 0.0 = show leading frame only (transition start)
- 0.5 = midpoint of transition (50% blended)
- 1.0 = show trailing frame only (transition end)

This enables smooth scrubbing in the timeline and variable-speed playback.

**Selective Support**

Not all transition types can be implemented efficiently with CIFilter:
- ✅ **Crossfade** - Native `CIDissolveTransition`
- ✅ **Fade to color** - `CIColorMatrix` with color overlay
- ✅ **Wipe** - `CISwipeTransition` with angle
- ❌ **Iris** - Requires custom Metal shader (deferred)
- ❌ **Blinds** - Requires custom Metal shader (deferred)

Unsupported types return `nil`, allowing the UI to display a placeholder message.

## Components

### 1. TransitionRenderer

**File:** `Sources/native-macos/Transitions/Rendering/TransitionRenderer.swift`

**Responsibilities:**
- Apply transition effect between two frames
- Manage filter compilation cache
- Calculate progress from timeline position
- Handle errors gracefully

**Interface:**

```swift
@MainActor
final class TransitionRenderer: Sendable {
    private let ciContext: CIContext
    private var filterCache: [TransitionType: Any] = [:]

    /// Apply transition effect between two frames
    /// - Parameters:
    ///   - leadingFrame: Frame from clip before transition
    ///   - trailingFrame: Frame from clip after transition
    ///   - transition: Transition configuration
    ///   - progress: Animation progress (0.0 = start, 1.0 = end)
    /// - Returns: Blended frame, or nil if transition unsupported
    func applyTransition(
        from leadingFrame: CIImage,
        to trailingFrame: CIImage,
        transition: TransitionClip,
        progress: Double
    ) -> CIImage?

    /// Clear cached filters (call when configuration changes)
    func invalidateCache()
}
```

**Progress Calculation:**

For a transition between time T1 and T2:
- Given current playback time `currentTime`
- Leading clip is active for `currentTime < T1 + transition.duration`
- Trailing clip is active for `currentTime > T1`

```
progress = (currentTime - T1) / transition.duration
Clamped to [0.0, 1.0]
```

### 2. CIFilter Extensions

**File:** `Sources/native-macos/Transitions/Rendering/CIFilterExtensions.swift`

**Responsibilities:**

**Interface:**

```swift
extension CIFilter {
```

**Implementation Notes:**

Each filter factory method:

2. Sets parameters from TransitionParameters
3. Validates required parameters are present
4. Returns nil on failure (invalid filter name, missing parameters)

## Technical Implementation Details

### Crossfade Implementation

**Filter:** `CIDissolveTransition`

**Parameters:**
- `inputImage`: Leading frame
- `inputTargetImage`: Trailing frame
- `inputTime`: Progress (0.0 to 1.0)

**Mapping:**
```swift
let filter = CIFilter(name: "CIDissolveTransition")
filter?.setValue(leadingFrame, forKey: kCIInputImageKey)
filter?.setValue(trailingFrame, forKey: kCIInputTargetImageKey)
filter?.setValue(progress, forKey: kCIInputTimeKey)
```

### Fade to Color Implementation

**Filter:** `CIColorMatrix`

**Approach:**
1. Fade leading frame to color (progress: 0.0 → 0.5)
2. Hold at color (holdDuration)
3. Fade from color to trailing frame (progress: 0.5 → 1.0)

**Mapping:**
```swift
let filter = CIFilter(name: "CIColorMatrix")
filter?.setValue(leadingFrame, forKey: kCIInputImageKey)

// Calculate RGB fade
let fadeProgress = min(progress * 2, 1.0) // 0.0 to 0.5
let r = color.redComponent * fadeProgress
let g = color.greenComponent * fadeProgress
let b = color.blueComponent * fadeProgress
filter?.setValue(CIVector(x: r, y: g, z: b, w: 0), forKey: "inputRVector")
filter?.setValue(CIVector(x: r, y: g, z: b, w: 0), forKey: "inputGVector")
filter?.setValue(CIVector(x: r, y: g, z: b, w: 0), forKey: "inputBVector")
filter?.setValue(CIVector(x: 0, y: 0, z: 0, w: 1 - fadeProgress), forKey: "inputAVector")
```

### Wipe Implementation

**Filter:** `CISwipeTransition`

**Parameters:**
- `inputImage`: Leading frame
- `inputTargetImage`: Trailing frame
- `inputAngle`: Wipe direction in degrees
- `inputExtent`: Output extent
- `inputTime`: Progress (0.0 to 1.0)

**Direction Mapping:**
```swift
let angle: Double
switch direction {
case .left: angle = 180
case .right: angle = 0
case .up: angle = 90
case .down: angle = 270
case .custom(let a): angle = a
}

filter?.setValue(angle, forKey: kCIInputAngleKey)
```

### Filter Cache Strategy

**Cache Key:** `TransitionType` (not full parameters)

**Rationale:**
- Transition type determines the filter class
- Parameters are set per-frame (fast)
- Filter compilation is expensive (cache this)

**Implementation:**
```swift
private func getFilter(for type: TransitionType) -> CIFilter? {
    if let cached = filterCache[type] {
        return cached as? CIFilter
    }

    guard let filter = createFilter(for: type) else {
        return nil
    }

    filterCache[type] = filter
    return filter
}
```

**Invalidation:**
```swift
func invalidateCache() {
    filterCache.removeAll()
}
```

Call when:
- Transition type changes
- Configuration parameters change significantly
- Memory warning received

## Error Handling

### Unsupported Transition Types

**Return `nil`** for:
- Iris transitions
- Blinds transitions
- Custom transitions
- Invalid filter names
- Missing required parameters

**UI Response:**
Show placeholder in transition position:
```swift
if let rendered = renderer.applyTransition(...) {
    // Show rendered frame
} else {
    // Show "Requires Metal shaders" placeholder
}
```

### Invalid Parameters

**Validation:**
- Progress clamped to [0.0, 1.0]
- RGB values clamped to [0.0, 1.0]
- Angles normalized to [0, 360)

**Fallback:**
Return leading frame if filter creation fails, allowing timeline to continue.

## Performance Considerations

### Target Frame Rate

**60fps** for smooth scrubbing and playback:
- ~16.67ms per frame budget
- CIFilter application: ~1-5ms (GPU-accelerated)
- CIContext rendering: ~5-10ms
- Total: ~6-15ms per frame (within budget)

### Adaptive Quality

**Not implemented in this phase** (deferred to Metal shader phase).

Current approach:
- Always render at full resolution
- Let CoreImage handle GPU optimization
- Monitor frame rate in testing

### Memory Management

**CIContext Lifecycle:**
- Create once, reuse for all renders
- Use GPU-accelerated context when available
- Release in `deinit`

**Filter Cache:**
- Store compiled CIFilter objects
- Clear on memory warning
- Typical size: 3 filters × ~1KB each = ~3KB

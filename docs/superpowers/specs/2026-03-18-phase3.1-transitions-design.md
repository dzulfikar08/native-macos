# Phase 3.1: Video Transitions - Design Specification

**Status:** Design Complete
**Date:** 2026-03-18
**Phase:** 3.1 (Video Transitions)

## Overview

Phase 3.1 implements a comprehensive video transition system for the OpenScreen video editor. The system supports multiple transition types (crossfade, wipes, iris, blinds), preset-based customization, real-time GPU-accelerated preview, and seamless export integration.

## Goals

1. **Flexibility:** Support multiple content types (narrative, social media, professional)
2. **Ease of Use:** Preset-based system with customization options
3. **Performance:** 60fps real-time preview with adaptive quality
4. **Extensibility:** Clean architecture for adding new transition types
5. **Professional Quality:** High-quality export with transitions baked in

## Architecture

**Approach:** Incremental Layer-Based

Transitions are implemented as a separate layer system that sits on top of the existing clip model without modifying VideoClip directly. This provides:
- Clean separation from existing clip logic
- Easier testing and maintenance
- No coupling between transitions and clips
- Simple extension for new transition types

### Data Models

#### TransitionClip

```swift
struct TransitionClip: Identifiable, Equatable, Codable, Sendable {
    let id: UUID
    var type: TransitionType
    var duration: CMTime
    let leadingClipID: UUID
    let trailingClipID: UUID
    var parameters: TransitionParameters
    var isEnabled: Bool

    init(
        id: UUID = UUID(),
        type: TransitionType,
        duration: CMTime,
        leadingClipID: UUID,
        trailingClipID: UUID,
        parameters: TransitionParameters,
        isEnabled: Bool = true
    )
}
```

**Properties:**
- `id`: Unique identifier
- `type`: Transition type (crossfade, wipe, iris, etc.)
- `duration`: Transition duration (0.1-5 seconds, limited by clip overlap)
- `leadingClipID`: Reference to clip before transition
- `trailingClipID`: Reference to clip after transition
- `parameters`: Type-safe transition parameters
- `isEnabled`: Enable/disable without removing

#### TransitionType

```swift
enum TransitionType: String, Codable, Sendable {
    case crossfade
    case fadeToColor
    case wipe
    case iris
    case blinds
    case custom(String)
}
```

**Built-in types:**
- `.crossfade` - Dissolves between clips
- `.fadeToColor` - Fade to black/white/custom color
- `.wipe` - Directional wipes (horizontal, vertical, diagonal)
- `.iris` - Shape-based reveals (circle, box, star)
- `.blinds` - Vertical/horizontal blinds
- `.custom(String)` - Extensible for custom shaders

#### TransitionValidator

```swift
struct TransitionValidator {
    func validate(_ transition: TransitionClip, availableOverlap: CMTime) throws {
        // Validate duration doesn't exceed overlap
        guard transition.duration <= availableOverlap else {
            throw TransitionError.durationExceedsOverlap
        }

        // Validate parameters
        guard transition.parameters.isValid else {
            throw TransitionError.invalidParameters
        }
    }
}
```

#### TransitionError

```swift
enum TransitionError: LocalizedError {
    case durationExceedsOverlap
    case invalidParameters
    case clipsNotFound
    case insufficientOverlap
    case parameterOutOfRange(String)

    var errorDescription: String? {
        // Error messages
    }
}
```

    let composition = AVMutableVideoComposition()

        // Add clip instruction

func transitionFilter(for type: TransitionType, parameters: TransitionParameters) -> CIFilter? {

## Known Limitations

1. **Transition overlap** - Transitions cannot overlap (Phase 3.2)
2. **Audio transitions** - Only crossfade, no advanced audio transitions (Phase 3.4)
3. **Custom shaders** - API exists but no editor UI (Phase 3.2)
4. **3D transitions** - No 3D perspective or depth effects
5. **Transition templates** - No animated transition templates (Phase 3.3)

# Phase 3.1.6: Export Integration - Design Specification

> **Status:** Design Approved
> **Created:** 2026-03-23
> **Phase:** 3.1.6 (Video Transitions - Export Integration)

## Overview

Implement AVFoundation export integration for video transitions, enabling users to export timelines with transitions baked into the final video file. This phase bridges the gap between real-time preview (Phase 3.1.3) and production-ready exports.

## Goals

- **Export quality:** Exported video matches preview quality frame-for-frame
- **All transition types:** Support crossfade, fade to color, wipe, iris, and blinds
- **Quality presets:** Provide draft/good/best presets with custom option
- **Fail-fast errors:** Export fails immediately with clear error messages
- **Performance:** Export time < 2× real-time for 1080p footage

## Architecture

### Design Philosophy

**Custom Compositor Pattern**

Following the established `PipCompositor` pattern in the codebase, Phase 3.1.6 implements `AVVideoCompositing` protocol to integrate transition rendering with AVFoundation's export pipeline. This approach:

- Reuses existing `TransitionRenderer` implementations from Phase 3.1.3
- Ensures exported video matches preview quality
- Supports all 5 transition types (not limited to simple opacity ramps)
- Keeps preview and export concerns separate (clean architecture)

**Bridge Pattern**

`TransitionVideoCompositor` acts as a bridge between AVFoundation's export system and our CoreImage-based renderers:

1. AVFoundation calls `startRequest(_:)` for each frame during export
2. Compositor extracts transition metadata from `TransitionCompositionInstruction`
3. Compositor fetches source frames from leading/trailing tracks
4. Compositor delegates rendering to appropriate `TransitionRenderer`
5. Compositor returns rendered frame to AVFoundation

**Quality Tier Strategy**

Three built-in quality presets plus custom option:

- **Draft:** 720p (1280×720), fast export, lower quality
- **Good:** 1080p (1920×1080), balanced speed/quality
- **Best:** Source resolution, highest quality, slower export
- **Custom:** User-configured resolution, bitrate, anti-aliasing

### 2. TransitionCompositionInstruction

**File:** `Sources/native-macos/Transitions/Export/TransitionCompositionInstruction.swift`

**Responsibilities:**
- Store transition metadata for AVFoundation instructions
- Pass transition configuration to compositor during export
- Create `AVVideoCompositionInstruction` instances with custom metadata

**IMPORTANT:** Cannot subclass `AVMutableVideoCompositionInstruction` (it's final in Objective-C runtime). Instead, use a wrapper class that creates instructions and stores metadata separately.

**Interface:**

```swift
/// Stores transition metadata and creates AVVideoCompositionInstruction instances
final class TransitionCompositionInstruction {

    // Store transition ID so compositor can look up full TransitionClip
    let transitionID: UUID

    // Store transition type and parameters for quick access without lookup
    let transitionType: TransitionType
    let transitionParameters: TransitionParameters

    // Transition timing for progress calculation
    let transitionStart: CMTime
    let transitionDuration: CMTime

    // Track IDs for fetching source frames
    let leadingTrackID: CMPersistentTrackID
    let trailingTrackID: CMPersistentTrackID

    // Time range for this instruction
    let timeRange: CMTimeRange

    init(
        transitionID: UUID,
        transitionType: TransitionType,
        transitionParameters: TransitionParameters,
        transitionStart: CMTime,
        transitionDuration: CMTime,
        leadingTrackID: CMPersistentTrackID,
        trailingTrackID: CMPersistentTrackID
    ) {
        self.transitionID = transitionID
        self.transitionType = transitionType
        self.transitionParameters = transitionParameters
        self.transitionStart = transitionStart
        self.transitionDuration = transitionDuration
        self.leadingTrackID = leadingTrackID
        self.trailingTrackID = trailingTrackID
        self.timeRange = CMTimeRange(start: transitionStart, end: CMTimeAdd(transitionStart, transitionDuration))
    }

    /// Creates the AVVideoCompositionInstruction for AVFoundation
    /// - Returns: Configured instruction with metadata stored in userInfo
    func makeAVInstruction() -> AVVideoCompositionInstruction {
        let instruction = AVMutableVideoCompositionInstruction()
        instruction.timeRange = timeRange
        instruction.enablePostProcessing = true

        // Store metadata in userInfo for compositor to retrieve
        instruction.userInfo = [
            "transitionID": transitionID,
            "transitionType": transitionType.rawValue,
            "transitionStart": NSValue(cmTime: transitionStart),
            "transitionDuration": NSValue(cmTime: transitionDuration),
            "leadingTrackID": NSNumber(value: leadingTrackID),
            "trailingTrackID": NSNumber(value: trailingTrackID)
        ] as [String: Any]

        return instruction
    }
}
```

**Usage in AVVideoCompositionBuilder:**

```swift
let instructionWrapper = TransitionCompositionInstruction(
    transitionID: transition.id,
    transitionType: transition.type,
    transitionParameters: transition.parameters,
    transitionStart: transitionStart,
    transitionDuration: transition.duration,
    leadingTrackID: leadingTrackID,  // Determined during composition build
    trailingTrackID: trailingTrackID // Determined during composition build
)

let avInstruction = instructionWrapper.makeAVInstruction()
instructions.append(avInstruction)
```

**Design Rationale:**

Storing `transitionID` enables compositor to look up full `TransitionClip` from `EditorState` for complex scenarios. Storing `transitionType` and `transitionParameters` directly avoids the lookup overhead for common cases.

### 5. Architecture: Two-Layer Builder Pattern

**IMPORTANT:** The export system uses two builder classes with distinct responsibilities:

**Layer 1: ExportCompositionBuilder** (`Sources/native-macos/Editing/ExportCompositionBuilder.swift`)
- **Responsibility:** Validate and orchestrate export composition building
- **Called by:** `TransitionExportPipeline`
- **Uses:** `AVVideoCompositionBuilder` internally
- **Current signature:** `buildForExport(from editorState: EditorState) async throws -> AVVideoComposition`

**Layer 2: AVVideoCompositionBuilder** (`Sources/native-macos/Transitions/Rendering/AVVideoCompositionBuilder.swift`)
- **Responsibility:** Build AVVideoComposition with transitions from EditorState
- **Called by:** `ExportCompositionBuilder`
- **Current signature:** `buildComposition(for editorState: EditorState) throws -> AVVideoComposition?`

**Quality Settings Flow:**
```
TransitionExportPipeline.export(to:quality:)
    ↓
ExportCompositionBuilder.buildForExport(from:quality:)  ← ADD quality parameter
    ↓
AVVideoCompositionBuilder.buildComposition(for:quality:)  ← ADD quality parameter
    ↓
Returns AVVideoComposition with custom compositor
```

4. **Track ID Management - CRITICAL:**

   **Problem:** `VideoClip` doesn't have an `assetTrackID` property. Track IDs are assigned during AVAsset composition creation, not stored on clips.

   **Solution:** During composition building, we create a mapping from clip IDs to track IDs:

   ```swift
   func buildComposition(
       for editorState: EditorState,
       quality: ExportQualitySettings? = nil
   ) throws -> AVVideoComposition? {
       // Get video tracks only
       let videoTracks = editorState.clipTracks.filter { $0.type == .video }
       guard !videoTracks.isEmpty else {
           return nil
       }

       // Create track ID mapping for each clip
       // Track IDs are 1-based integers used by AVFoundation
       var clipTrackIDs: [UUID: CMPersistentTrackID] = [:]
       for (trackIndex, track) in videoTracks.enumerated() {
           let trackID = CMPersistentTrackID(trackIndex + 1)
           for clip in track.clips {
               clipTrackIDs[clip.id] = trackID
           }
       }

       // Flatten clips from all video tracks in time order
       let allClips = videoTracks
           .flatMap { $0.clips }
           .sorted { $0.timeRangeInTimeline.start < $1.timeRangeInTimeline.start }

       guard !allClips.isEmpty else {
           return nil
       }

       // Build instructions for clips and transitions
       var instructions: [AVVideoCompositionInstruction] = []

       for clip in allClips {
           let clipTrackID = clipTrackIDs[clip.id]!

           // Check for transitions involving this clip
           let outgoingTransitions = editorState.transitions.filter {
               $0.leadingClipID == clip.id
           }

           let incomingTransitions = editorState.transitions.filter {
               $0.trailingClipID == clip.id
           }

           // Add instruction for clip portion (not including transition)
           // ... existing clip instruction logic ...

           // Add transition instructions
           for transition in outgoingTransitions {
               guard let trailingClipID = transition.trailingClipID,
                     let trailingTrackID = clipTrackIDs[trailingClipID] else {
                   throw TransitionError.clipsNotFound(
                       leadingClipID: nil,
                       trailingClipID: transition.trailingClipID
                   )
               }

               let instructionWrapper = try buildTransitionInstruction(
                   transition: transition,
                   leadingClip: clip,
                   leadingTrackID: clipTrackID,
                   trailingClip: findClip(id: trailingClipID, in: editorState),
                   trailingTrackID: trailingTrackID,
                   editorState: editorState
               )

               // Convert wrapper to AVVideoCompositionInstruction
               let avInstruction = instructionWrapper.makeAVInstruction()
               instructions.append(avInstruction)
           }
       }

       // ... rest of method ...
   }

   private func findClip(id: UUID, in editorState: EditorState) -> VideoClip? {
       return editorState.clipTracks
           .flatMap { $0.clips }
           .first { $0.id == id }
   }
   ```

   **Key Points:**
   - Track IDs are assigned sequentially starting from 1 (CMPersistentTrackID)
   - Each clip gets the track ID of its parent track
   - Track IDs are consistent between clip instructions and transition instructions
   - Transition instructions reference the track IDs of their leading and trailing clips

5. **AVAsset Composition - IMPORTANT:**

   **Current Gap:** The existing `AVVideoCompositionBuilder` works with `EditorState` but doesn't create the `AVAsset` or `AVComposition` with tracks. The track IDs we assign assume an AVAsset will be created elsewhere.

   **Where AVAsset is Created:** Looking at the existing codebase, `VideoExporter.init(asset:outputURL:)` receives an `AVAsset` parameter. This asset must be created before export begins.

   **Two Approaches:**

   **Option A (Recommended):** Asset created by `TransitionExportPipeline` before calling builders
   ```swift
   func export(to outputURL: URL, quality: ExportQualitySettings) async throws {
       // Create AVAsset composition from EditorState
       let asset = try buildAVAsset(from: editorState)

       // Build video composition (which references track IDs from asset)
       let composition = try await compositionBuilder.buildForExport(
           from: editorState,
           quality: quality,
           asset: asset  // Pass asset so track IDs match
       )

       let exporter = VideoExporter(
           asset: asset,
           outputURL: outputURL,
           quality: quality
       )
       // ... rest of export
   }
   ```

   **Option B:** Asset created inside `AVVideoCompositionBuilder`
   - More cohesive (all composition logic in one place)
   - But creates tight coupling between builder and asset creation

   **Decision:** Use Option A. Keep asset creation separate from video composition building for cleaner separation of concerns.

   ```swift
   /// - Throws: TransitionError if asset cannot be built
   private func buildAVAsset(from editorState: EditorState) throws -> AVMutableComposition {
       let composition = AVMutableComposition()

       // Get video tracks
       let videoTracks = editorState.clipTracks.filter { $0.type == .video }
       guard !videoTracks.isEmpty else {
           throw ExportError.noVideoTracks

       // Add each clip's asset to the composition

               let assetTrack = clip.asset.tracks(withMediaType: .video).first!
               try compositionTrack?.insertTimeRange(

                   of: assetTrack,

       }
   ```

## Data Flow

### Export Pipeline Flow

```
User clicks Export in UI
    ↓
Export dialog shows quality preset selector
User selects quality (Draft/Good/Best/Custom)
    ↓
TransitionExportPipeline.export(to: quality:)
    ↓
validateTransitions() - Verify all transitions have source clips
    ↓
ExportCompositionBuilder.buildForExport(from: quality:)
    ↓
AVVideoCompositionBuilder.buildComposition(for: quality:)
    ├─ Build clip instructions (existing logic)
    └─ Build TransitionCompositionInstruction for each transition
        ├─ Extract track IDs from AVAsset composition
        ├─ Calculate transition timing (start, duration)
        └─ Store transition metadata (type, parameters)
    ↓
Set composition.customVideoCompositorClass = TransitionVideoCompositor.self
    ↓
VideoExporter.startExport()
    ↓
AVAssetExportSession starts export
    ↓
[For each frame during export]
AVFoundation calls TransitionVideoCompositor.startRequest(request:)
    ├─ Extract transitionID from request.compositionInstruction
    ├─ Look up TransitionClip from EditorState
    ├─ Get source buffers from leading/trailing tracks:
    │   leadingBuffer = request.sourceFrame(byTrackID: leadingTrackID)
    │   trailingBuffer = request.sourceFrame(byTrackID: trailingTrackID)
    ├─ Calculate progress:
    │   elapsed = request.compositionTime - transitionStart
    │   progress = elapsed.seconds / transitionDuration.seconds
    │   progress = clamp(progress, 0.0, 1.0)
    ├─ Get renderer from TransitionRenderContext
    ├─ Call renderer.render(sourceBuffer, targetBuffer, progress, transition)
    └─ request.finish(withComposedVideoFrame: renderedBuffer)
    ↓
Export completes successfully
    ↓
User receives output file with transitions baked in
```

### Progress Calculation Detail

```swift
func calculateProgress(
    currentTime: CMTime,
    transitionStart: CMTime,
    transitionDuration: CMTime
) -> Double {
    let elapsed = CMTimeSubtract(currentTime, transitionStart)
    let elapsedSeconds = CMTimeGetSeconds(elapsed)
    let durationSeconds = CMTimeGetSeconds(transitionDuration)
    var progress = elapsedSeconds / durationSeconds

    // Clamp to [0.0, 1.0]
    progress = max(0.0, min(1.0, progress))

    return progress
}
```

**Linear Interpolation:**
- Progress 0.0: Show leading clip frame only
- Progress 0.5: 50% blend (midpoint of transition)
- Progress 1.0: Show trailing clip frame only

This matches preview behavior exactly for frame-perfect exports.

## Error Handling

### Fail-Fast Strategy

Per our design choice, export fails immediately when transition rendering fails. This ensures:

- No partially-corrupted output files
- User knows exactly what went wrong
- Can fix timeline and retry
- No silent quality degradation

### Error Types

**1. Missing Source Clips**

```swift
case .clipsNotFound(leadingClipID: UUID?, trailingClipID: UUID?)
```

**When:** Leading or trailing clip deleted from timeline after transition created

**Handling:**
- Validation catches this before export starts
- Export fails with clear message: "Transition 'Crossfade' references missing clip [clip-id]"
- User must delete transition or restore clip

**2. Renderer Failures**

```swift
case .invalidParameters(reason: String)
case .renderingFailed(underlying: Error)
```

**When:** TransitionRenderer throws during export (e.g., CoreImage filter creation fails)

**Handling:**
- Compositor catches error in `startRequest(_:)`
- Calls `request.finish(with: error)`
- AVFoundation fails export and propagates error
- User sees: "Export failed: Failed to render transition 'Wipe' - invalid direction parameter"

**3. Pixel Buffer Failures**

```swift
case .pixelBufferAllocationFailed
```

**When:** Out of memory or invalid pixel format

**Handling:**
- Renderer throws when CVPixelBufferCreate fails
- Export fails immediately
- User sees: "Export failed: Out of memory during transition render"

## Performance Considerations

### Export Performance Target

**Goal:** < 2× real-time for 1080p footage

- 60-second video with 5 transitions should export in < 120 seconds
- Assumes modern Mac (M1/M2 or equivalent)

**Factors affecting performance:**
1. **Resolution** - 720p fastest, 4K slowest
2. **Transition count** - More transitions = more rendering work
3. **Transition complexity** - Iris/blinds slower than crossfade
4. **Anti-aliasing** - Multi-sample AA adds 30-50% overhead

```swift
    private var pixelBufferPool: CVPixelBufferPool?

    func createPixelBuffer(size: CGSize) throws -> CVPixelBuffer {
        if let pool = pixelBufferPool,
           let buffer = pool.createPixelBuffer() {
            return buffer
        }

        // Fall back to allocation
        return try allocatePixelBuffer(size: size)
    }
}
```

**2. Render Caching** (Future Enhancement)

Cache rendered frames for transitions at standard progress values (0.0, 0.25, 0.5, 0.75, 1.0) to avoid redundant rendering.

**3. Parallel Rendering** (Future Enhancement)

For export with multiple independent transitions, render in parallel using `Task` concurrency.

### Memory Management

**Pixel Buffer Lifecycle:**

```swift
func startRequest(_ request: AVAsynchronousVideoCompositionRequest) {
    // Source buffers owned by AVFoundation
    let leadingBuffer = request.sourceFrame(byTrackID: leadingTrackID)
    let trailingBuffer = request.sourceFrame(byTrackID: trailingTrackID)

    // Output buffer created by renderer
    let outputBuffer = try renderer.render(
        sourceBuffer: leadingBuffer,
        targetBuffer: trailingBuffer,
        progress: progress,
        transition: transition
    )

    // Transfer ownership to AVFoundation
    request.finish(withComposedVideoFrame: outputBuffer)
}
```

**Memory Safety:**
- Source buffers not retained (owned by AVFoundation)
- Output buffer transferred to AVFoundation via `finish(withComposedVideoFrame:)`
- No retain cycles (compositor is `@MainActor` value type)

## Known Limitations

1. **Custom transition shaders** - API exists but no editor UI for creating custom shaders (deferred to Phase 3.2)

2. **Audio transitions** - Only crossfade, no advanced audio transitions (deferred to Phase 3.4)

3. **Real-time preview during export** - No live preview of export progress (export progress shows percentage only)

4. **Hardware acceleration** - CoreImage uses GPU, but custom compositor adds CPU overhead for coordination

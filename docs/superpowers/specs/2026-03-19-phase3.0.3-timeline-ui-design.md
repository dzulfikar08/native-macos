# Phase 3.0.3: Timeline UI and Rendering - Design Specification

**Date:** 2026-03-19
**Status:** Design Approved
**Phase:** Timeline UI and Rendering
**Dependencies:** Phase 3.0.1 (Foundation), Phase 3.0.2 (Undo/Redo)

## Overview

Phase 3.0.3 builds the visual user interface for multi-clip video editing on the timeline. This phase implements a SwiftUI-based timeline with clip visualization, drag-to-reposition functionality, advanced snapping, and bidirectional synchronization with the video preview.

**Scope:** Timeline visualization with clips and tracks, plus basic drag-to-reposition functionality. No advanced editing like ripple edits or split operations in this phase.

## Architecture

### Approach: Hybrid SwiftUI with Smart Coordinator

The architecture uses SwiftUI for declarative UI and built-in virtualization, while extracting complex logic (snapping, drag state machine, caching) into focused coordinator types.

**Three-Layer Architecture:**
1. **View Layer** - SwiftUI views for rendering (TimelineView, ClipTrackView, ClipView, etc.)
2. **Coordinator Layer** - Business logic coordinators (TimelineViewModel, SnappingCoordinator, UndoRedoCoordinator, ClipLayoutCache)
3. **Data Layer** - Existing Phase 3.0.1/3.0.2 models (EditorState, ClipManager, ClipUndoManager, VideoClip, ClipTrack)

**Data Flow:**
- **Downward:** EditorState → TimelineViewModel → TimelineView → child views
- **Upward:** User interactions → TimelineView → TimelineViewModel → ClipUndoManager → EditorState

## Components

### TimelineViewModel

**Purpose:** Bridge between UI and data layers, manages UI-specific state, coordinates helper coordinators.

**Type:** `@Observable` class (Swift 5.9 observation framework)

**Properties:**
- `timelineEditMode: TimelineEditMode` - Computed from EditorState
- `tracks: [ClipTrack]` - Computed from EditorState.clipTracks
- `selectedClipIDs: Set<UUID>` - Currently selected clips
- `playheadPosition: CMTime` - Current playhead position
- `isDragging: Bool` - Whether a drag is in progress
- `draggedClip: VideoClip?` - The clip being dragged
- `dragOffset: CGSize` - Current drag offset from original position
- `snapIndicator: SnapIndicator?` - Visual indicator for snap (position, type)
- `visibleTimeRange: CMTimeRange` - Currently visible time range (for virtualization)

**Coordinator References:**
- `snappingCoordinator: SnappingCoordinator` - Handles snap calculations
- `undoRedoCoordinator: UndoRedoCoordinator` - Wraps operations for undo/redo
- `layoutCache: ClipLayoutCache` - Caches clip positions

**Methods:**
- `selectClip(_ clipID: UUID)` - Select a clip
- `selectClips(_ clipIDs: [UUID])` - Select multiple clips
- `deselectAll()` - Clear selection
- `startDrag(clip: VideoClip, at: CGPoint)` - Begin dragging a clip
- `updateDrag(at: CGPoint)` - Update drag position, handles snapping
- `endDrag()` - Commit drag operation, creates MoveClipOperation
- `cancelDrag()` - Cancel drag, revert to original position
- `setPlayheadPosition(_ position: CMTime)` - Update playhead, syncs with video preview
- `handlePlaybackAction(_ action: PlaybackAction)` - Play/pause/stop

**Notifications Posted:**
- `.timelineClipSelectionChanged` - When selection changes
- `.timelineClipDragging` - During drag (with position updates)
- `.timelineDragEnded` - When drag completes
- `.timelinePlayheadMoved` - When playhead position changes
- `.timelineDragCancelled` - When drag is cancelled

### SnappingCoordinator

**Purpose:** Calculate snap points and apply snap offsets during drag operations.

**Type:** Plain Swift class (no UI dependencies for testability)

**Properties:**
- `snapTolerance: TimeInterval` - Time distance within which snapping activates (default: 0.5s)
- `snapToClipEdges: Bool` - Snap to clip start/end (default: true)
- `snapToTrackBoundaries: Bool` - Snap to track boundaries (default: true)
- `snapToPlayhead: Bool` - Snap to playhead position (default: true)
- `snapToTimeIncrements: Bool` - Snap to time increments (default: true)
- `timeIncrement: TimeInterval` - Time increment to snap to (default: 1.0s)
- `snapToMarkers: Bool` - Snap to markers (default: true)

**Methods:**
- `calculateSnapPoints(for clip: VideoClip, in tracks: [ClipTrack], excluding: Set<UUID>) -> [SnapPoint]`
- `findSnapPoint(for position: CMTime, snapPoints: [SnapPoint]) -> SnapResult?`
- `applySnapOffset(to position: CMTime, snapResult: SnapResult) -> CMTime`

**SnapPoint Type:**
```swift
enum SnapPointType {
    case clipEdge          // Start or end of another clip
    case trackBoundary     // Track start position
    case playhead          // Current playhead position
    case timeIncrement     // Regular time interval (e.g., every second)
    case marker            // Chapter marker position
}

struct SnapPoint {
    let position: CMTime
    let type: SnapPointType
    let source: String?    // Optional description (e.g., "Clip 'Intro' start")
}
```

**SnapResult Type:**
```swift
struct SnapResult {
    let snapPoint: SnapPoint
    let originalPosition: CMTime
    let snappedPosition: CMTime
    let offset: CMTime
}
```

### ClipLayoutCache

**Purpose:** Cache clip position calculations for performance, track dirty state.

**Type:** Plain Swift class

**Properties:**
- `layoutCache: [UUID: ClipLayout]` - Cache of clip layout data keyed by clip ID
- `tracksCacheHash: [TrackType: Int]` - Hash of each track's clips to detect changes
- `viewportWidth: CGFloat` - Current viewport width for coordinate conversion
- `timeScale: CGFloat` - Pixels per second (zoom level)

**ClipLayout Type:**
```swift
struct ClipLayout {
    let clipID: UUID
    let frame: CGRect
    let timeRange: CMTimeRange
    var isDirty: Bool
}
```

**Methods:**
- `invalidate(clipID: UUID)` - Mark clip as dirty
- `invalidate(track: ClipTrack)` - Mark all clips in track as dirty
- `invalidateAll()` - Mark all clips as dirty
- `updateLayout(for clip: VideoClip, in track: ClipTrack, viewportWidth: CGFloat, timeScale: CGFloat) -> CGRect`
- `setTimeScale(_ scale: CGFloat)` - Update zoom level, invalidates all layouts
- `cleanup(_ clipIDs: Set<UUID>)` - Remove cache entries for deleted clips

**Coordinate Conversion:**
- `timeToX(_ time: CMTime, scale: CGFloat) -> CGFloat` - Convert time to x-position
- `xToTime(_ x: CGFloat, scale: CGFloat) -> CMTime` - Convert x-position to time
- `durationToWidth(_ duration: CMTime, scale: CGFloat) -> CGFloat` - Convert duration to width

**Cache Invalidation Triggers:**
- Clips added/removed from tracks
- Clip time ranges change
- Viewport width changes (window resize)
- Time scale changes (zoom in/out)

### UndoRedoCoordinator

**Purpose:** Bridge between timeline UI interactions and ClipUndoManager from Phase 3.0.2.

**Type:** Plain Swift class

**Properties:**
- `undoManager: ClipUndoManager` - Reference to the undo manager
- `isUndoingOrRedoing: Bool` - Flag to prevent recursive operations

**Methods:**
- `executeMoveOperation(clip: VideoClip, newTimeRange: CMTimeRange, newTrackID: UUID?)`
- `executeTrimOperation(clip: VideoClip, newTimeRange: CMTimeRange)` - Future phase
- `executeSplitOperation(clip: VideoClip, at time: CMTime)` - Future phase
- `executeDeleteOperation(clips: [VideoClip], rippleEdits: Bool)` - Future phase
- `executeDuplicateOperation(clip: VideoClip, to timeRange: CMTimeRange, trackID: UUID)` - Future phase
- `executeChangeSpeedOperation(clip: VideoClip, newSpeed: Double)` - Future phase

**Phase 3.0.3 Scope:** Only `executeMoveOperation` is implemented for drag-to-reposition.

## Interactions

### Drag Gesture Handling

**Drag State Machine:**
1. **Idle** - No drag in progress
2. **DragStarted** - User pressed on clip, calculating initial position
3. **Dragging** - Clip is being dragged, snap calculations active
4. **Committing** - Drag released, creating and executing MoveClipOperation
5. **Cancelled** - Drag cancelled (e.g., escape key), reverting to original position

**Drag Gesture Implementation:**
```swift
DragGesture()
    .onChanged { value in
        viewModel.updateDrag(at: value.location)
    }
    .onEnded { value in
        viewModel.endDrag()
    }
```

**updateDrag Workflow:**
1. Convert gesture location to timeline time using `ClipLayoutCache.xToTime()`
2. Calculate new clip position based on drag offset
3. Call `SnappingCoordinator.findSnapPoint()` to check for snaps
4. If snap found within tolerance:
   - Adjust position to snap target
   - Set `snapIndicator` to show visual feedback
5. Update `dragOffset` to reflect snapped position
6. Post `.timelineClipDragging` notification

**endDrag Workflow:**
1. Calculate final time range from drag position
2. If position changed from original:
   - Create `MoveClipOperation` with new position
   - Call `UndoRedoCoordinator.executeMoveOperation()`
   - Post `.timelineClipMoved` notification
3. Clear drag state and snap indicator
4. Post `.timelineDragEnded` notification

**Visual Feedback During Drag:**
- Dragged clip has elevated shadow and slight opacity (0.8)
- Ghost clip shows original position (semi-transparent outline)
- Snap indicator line appears when snap activates
- Cursor changes to open/closed hand state
- Clip border highlights during drag

### Playhead Sync with Video Preview

**Bidirectional Synchronization:**

1. **Timeline → Preview:**
   - User drags playhead on timeline
   - `TimelineViewModel.setPlayheadPosition()` called
   - VideoPreview seeks to new position via VideoProcessor
   - Preview frame updates

2. **Preview → Timeline:**
   - During playback, VideoProcessor posts frame notifications
   - TimelineViewModel observes notifications
   - Updates `playheadPosition` property
   - Timeline view re-renders with new playhead position
   - Auto-scrolls to keep playhead visible

3. **Playback Controls:**
   - Play/Pause buttons in PlaybackControlsView
   - Calls `TimelineViewModel.handlePlaybackAction()`
   - Forwards to VideoProcessor to start/stop playback
   - Timeline playhead animates during playback

### Mode Switching

**Single-Asset Mode:**
- Timeline shows simplified view
- Single track representation (current trim range)
- No drag-to-reposition (use existing trim controls)
- Time ruler shows asset duration

**Multi-Clip Mode:**
- Timeline shows all tracks and clips
- Full clip visualization with thumbnails
- Drag-to-reposition enabled
- Track headers visible
- Advanced snapping active

**Mode Switch:**
- Handled by existing `SwitchTimelineModeOperation` from Phase 3.0.2
- Timeline observes `EditorState.timelineEditMode`
- View adapts rendering based on current mode
- Smooth transition animation

## Data Flow

### Clip Drag Operation Flow

1. User starts dragging → `DragGesture.onChanged` fires
2. `TimelineViewModel.updateDrag(at:)` called
3. New position calculated from gesture location
4. Snap check via `SnappingCoordinator.findSnapPoint()`
5. If snap found: adjust position, set `snapIndicator`
6. View model updates `dragOffset` and `snapIndicator`
7. SwiftUI observes view model changes → `ClipView` re-renders at new position
8. User releases drag → `DragGesture.onEnded` fires
9. `TimelineViewModel.endDrag()` called
10. `UndoRedoCoordinator.executeMoveOperation()` creates operation
11. `ClipUndoManager.executeOperation()` executes operation
12. Operation updates `EditorState` (modifies clip position in track)
13. `EditorState` posts `.clipTracksChanged` notification
14. `TimelineViewModel` invalidates cache for affected clips
15. Timeline re-renders with new positions

### Error Handling

**Error Types from ClipError:**
- **wouldOverlap** - Can't move clip to overlapping position
- **trackNotFound** - Target track doesn't exist
- **clipNotFound** - Clip was deleted during drag
- **operationFailed** - Generic operation failure

**Error Presentation:**
- TimelineViewModel observes error notifications
- Errors presented via `ErrorPresenter.presentError(_ error: ClipError)`
- Drag operation cancelled and reverted to original position
- User can retry after understanding the error

**Pre-Drag Validation:**
- Check if clip is locked or read-only
- Check if track is enabled (not muted/disabled)
- Validate clip has valid time range
- Check for conflicts before starting drag

## Performance Optimization

**Virtualization:**
- LazyVStack for track rendering (only visible tracks)
- LazyHStack for clip rendering (only visible clips in each track)
- Clips outside `visibleTimeRange` not rendered

**Caching:**
- ClipLayoutCache avoids recalculating clip frames on every render
- Thumbnail generation prioritized for visible clips
- Cache hit rate target: >90%

**Coordinate Conversion:**
- Pre-calculate time-to-x scale factors
- Cache viewport dimensions
- Minimize CMTime conversions

**Background Processing:**
- Thumbnail generation on background queue
- Waveform calculation on background queue
- Snap point calculations optimized to <16ms

**Performance Targets:**
- Render 100+ clips across 10+ tracks at 60fps
- Drag interactions respond within 16ms (one frame)
- Snap calculations complete in <5ms
- Cache invalidation only when necessary

## Limitations (By Design)

The following features are explicitly **out of scope** for Phase 3.0.3:

- No clip trimming (resize handles on clip edges)
- No clip splitting (cut clip at playhead)
- No clip deletion from timeline
- No clip duplication from timeline
- No ripple editing
- No speed changes from timeline
- No effect editing from timeline
- No marker creation from timeline
- No track creation/deletion from timeline
- No clip audio level adjustments
- No clip compositing mode changes
- No keyboard shortcuts for timeline operations

These features will be considered for future phases based on Phase 3.0.3 outcomes.
